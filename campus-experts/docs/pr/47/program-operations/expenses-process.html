Submitting Reimbursements for GitHub Education Projects via Open Collective

Github Education' Campus Experts partners with Open Source Collective (a 501c6 open source sustainability nonprofit) as its fiscal host and uses the Open Collective platform to process reimbursements for approved student projects.

Before You Begin

Review Open Source Collective’s Documentation Make sure you read through Open Source Collective’s ‘Invoices and Reimbursements’open in new window documentation to understand their process and what is required on reimbursements

Create an Open Collective Account

If you already have an Open Collective account, skip this part If you don’t have an account: Create a profileopen in new window.

Steps to Request Reimbursement

  • Head over to https://opencollective.com/github-education and follow the GitHub Education Collective.
  • After following, find the "Submit Expense" button
  • Next, select "Reimbursement" (For CE Events) from the two options.
  • Fill out the Payee information and add your bank details or your PayPal account
  • Once through, click "Next" and then fill out Expense details.
  • Upload all of the receipts as individual line items in the Expense.
  • Add the tag "ce-sponsorship" below "Tag your expense".
  • The remainder is straightforward: fill out the "Invoice details" section based on the receipts you're submitting, add notes if any are needed and you're all set to submit! It’s helpful to include a very short description of the type of event it was and how the purchases supported it.

You do not need to upload an invoice. The invoice is for GitHub purposes only. Please email your invoice to Pj at metzinaround@github.com. The only thing required for reimbursements is to upload all receipts for your purchases.

Payment Schedule

Payments are processed twice a week on Tuesdays and Thursdays at 8:00am Pacific Time.

If You Need Support

For issues with expenses, or if you’re having difficulty with receiving your payments, email hello@oscollective.org.

For account setup, password retrieval, or other platform-related inquiries, email support@opencollective.com

Filling out the invoice

  • Invoice templateopen in new window Make a copy of this and name it YourLastName_invoice_##### where ##### is the invoice number (see below for Invoice number instructions).
  • Create an invoice for the total amount, this invoice should include a list of all expenses and their amounts.

[!NOTE] IF YOU ARE FILLING THIS OUT FOR IOPEN COLECTIVE, PLEASE USE THE ADDRESS ABOVE.

  • Fill out the invoice with the following details, as shown in the screenshot:
    Attn: Pj Metz
    GitHub Inc,
    88 Colin P Kelly Jr Street,

    CA 94107,

    (This does not mean that you need to physically mail your invoice in, but digital invoices must also be addressed here)
  • For the Invoice number, this indicates how many invoices you have sent the program and it should start with your GitHub IDopen in new window. After your GitHub Id, please add a 001 to indicate which invoice number this is for you. The next time you submit an invoice it should be "002" and so on and so forth. For example, if Pj were to create an invoice for the first time, it would be numbered 65838556001. Pj's GitHub ID is 65838556.
  • The line items in the invoice do not need to say "Dates Worked." Rather, it should be formatted to say what the line item is for, and a total cost for that particular line item.
  • To get the total cost in the Invoice, work backwards from the total cost. If the pizza was $250, and the event was 5 hours, then the hourly rate should be $50. Inputting the hourly rate and the hours will automatically fill out the total at the end of the row.
  • Invoices are paid in 30 days, sometimes longer for international students.
