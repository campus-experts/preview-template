# Creating an issue for your event

When you host an event as a GitHub Campus Expert, the first step is to open an issue in the being-an-expertopen in new window repo. You don't need to be asking for support to make an issue in the repo - you're free to file a report even if you don't need support and just want to show us the cool stuff you're doing!

Each issue is read by a member of the Education Team, and reviewed by-hand, so you may not receive instant replies - but be assured, you will get a response!

# Creating an Issue

Creating an issue is relatively straightforward, head over to the Issuesopen in new window tab and hit the shiny green "New issue" button. You will then be prompted to use an "Event Issue" template.

You can now proceed to fill out the event issue. Make sure to fill out the forms!

# Labels

Before you submit, make sure you assign yourself to the issue, and add labels.
When adding labels, add ones that cover the following questions:

  • where you are
  • what you're doing
  • what support you need (optional)

# Support for Events

GitHub provides support for Campus Expert events in two ways:

  • Swag 👕
    • T-shirt Coupons
    • Mona Figurines
    • GitHub Stickers
    • Git Cheat Sheets
    • Comic Books
    • Markdown guides
    • GitHub Flow Guides
  • Monetary Support 💸
    • See Sponsorship Guidelines

Swag can always be requested for any event, as long as the event issue is made and reviewed 30 days in advance and you have completed the Post-Event reports for previous event(s), if any. Make sure you plan well in advance!

NOTE: The rule of thumb when requesting swag or support for your event is that you should have already filed a post-event report for your previous event. If you do not do so, you are not eligible to request swag for your current event. However, this is not a hard-and-fast rule, and each case may be different. If you can demonstrate a reason, GitHub Education Staff will give you the opportunity to do so, and take a call.

For information on monetary support, please read the sponsorship guidelines page.

# Example

🚩 Open an issue

I open an issue "GitHub Copilot Workshop at University of Nottingham", and describe what I'm planning to do according to the issue template.

⚠️ Add labels

I add the labels "UK" and "Workshop" to describe where I am and what I'm doing, and add "swag wanted" because I would like GitHub cheat sheets and stickers. (You don't need to create a label for topic, or city) Keep it simple: location: country request: sponsorship request / swag request etc.

That's it ✨

🤵🙋🏿 Tag anyone you think can help you

I tag @fellow-campus-expert because I know they are another Campus Expert nearby and might want to help because I think they can contribute to my workshop content and plan, and wait for my issue to be reviewed.

✨ Do the thing!

After getting the help I need, I hold the workshop.

👍 Tell us how it went

I come back to the issue to tell everyone how it went! I might include photos from the event, relevant social media posts and the number of attendees. I will create the post event report using the Post Event Templateopen in new window by copying the content of the template to my issue and updating it there (I do not change the main template!), and submit any expenses by following the Expense procedure.

✅ All done, thank you!

After posting the post-event report, I DON'T close the issue: someone from the team will close it after updating everything on their end.
